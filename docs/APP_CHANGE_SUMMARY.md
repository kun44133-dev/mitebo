# 密特堡压力监测 App 功能变更总结

更新时间：2026-06-17

## 一、项目目标

本项目最初目标是将 `iot.mitebo.cn` 网站能力改造成可安装、可登录、可操作的 Android App，而不是简单网页套壳。

后续逐步围绕设备监控、模具监控、网关管理、告警中心、压力单位、静止压力采集、后台告警提醒、UI 美化、图标资源、版本控制等方向持续改造。

## 二、当前技术栈

- Android 原生应用
- Java
- Gradle / Android Gradle Plugin
- Android SDK 35
- 最低支持 Android 6.0，`minSdk 23`
- HTTP 接口请求：`HttpURLConnection`
- UI：原生 View 代码动态构建
- 数据格式：JSON
- 本地存储：`SharedPreferences`
- 后台告警：Android `Service`
- 通知、角标、铃声、震动：Android 系统能力

- 上下限弹窗显示实时压力、静止压力、电量。
- 压力单位切换后，模具实时压力、静止压力、上下限数值同步换算并显示单位。

### 3. 静止压力采集

已做过多轮策略调整，当前核心方向：

- 不从曲线图抓取静止压力，避免加载慢。
- 静止压力以模具停机离线后的稳定压力为准。
- 模具在线生产时锁定静止压力，不允许动态压力覆盖。
- 模具下线后延迟并判断压力稳定，再更新静止压力。
- 静止压力按账号隔离，避免多个账号之间数据串用。
- 切换账号时重置压力状态缓存。
- 尽量避免离线模具静止压力长期显示异常默认值。

- 增加“添加监控”功能，可添加多个 MAC 进行监控。
- 添加监控设备按账号隔离，避免 A 账号添加的监控设备跑到 B 账号。
- 支持下拉刷新。
- 设备卡片支持左滑删除。

- 设备详情弹窗增加信息修改入口。
- 信息修改中支持上下限修改。
- 设备卡片左侧图标由绿点改为自定义压力传感器 SVG 图标。
- 传感器在线、离线、告警状态按网站压力变送器状态逻辑补充优化。
- 设备卡片布局压缩高度并优化排版。

### 5. 告警中心页面

已实现：

- 告警列表显示当天告警。
- 告警信息条显示：
  - 告警类型
  - 传感器
  - 车间 / 模具 / 气路等描述
  - 压力详情

- 告警状态：
  - 已消除：绿色
  - 未消除：红色
- 告警信息左侧图标改为铃铛：
  - 已消除：绿色铃铛
  - 正在告警：红色铃铛
- 告警列表支持伸缩详情。
- 告警详情底部按钮改为“上下限修改”。
- 多传感器告警时，上下限修改菜单按当前传感器数量加载传感器下拉菜单。
- 告警信息支持左滑删除。
- 手动删除告警后立即刷新列表和角标。
- 告警页面顶部按钮：
  - 今日全部
  - 未消除
  - 日期查询
- 日期查询支持选择日期显示告警。
- “今日全部”按钮双击切换：
  - 普通状态：显示当天产生的报警数量和当天报警列表。
  - 双击后：显示历史至今全部报警数量和全部报警列表，未消警排在前面。
  - 再次双击：回到今日报警。
- 修复今日报警数量错误显示为历史全部数量的问题。
- 当前今日报警数量按告警产生日期统计，不直接信任接口 `total`。
- 当前今日报警数量支持分页累计，每页 1000 条，超过 1000 条会继续拉取后续页统计，避免卡在 1000。

## 五、数据与接口策略

### 1. 接口来源

App 使用网站后端接口，例如：

- `/yujing/device/list`
- `/yujing/alarm/list`
- `/yujing/mould/list`
- `/yujing/gateway/list`
- `/yujing/mould/onList`
- `/yujing/gateway`
- `/yujing/device`

### 2. 刷新策略

- 模具页打开立即刷新。
- 网关页打开立即刷新。
- 模具页实时压力只刷新数值，尽量避免整页频繁刷新。
- 告警信息前台定时刷新。
- 后台告警监控开启后，只轮询告警，不刷新模具压力。
- 网络慢时增加请求版本校验，旧请求返回后不再覆盖新页面，避免重复卡片和错乱。

### 3. 多账号隔离

已做隔离：

- 多账号密码记忆。
- 监控设备列表按账号隔离。
- 静止压力状态按账号隔离。
- 切换账号重置压力状态缓存。

## 七、已知注意事项

1. 今日报警数量现在按告警产生日期分页统计，不再直接使用接口 `total`。
2. 历史全部报警数量仍使用接口返回的历史总数。
3. 如果某一天告警量极大，分页统计会增加接口请求次数，但能避免数量卡在 1000。
4. APK 体积变小是因为大图资源已移除，不代表功能缺失。
5. Android 桌面角标在不同手机系统上的支持效果可能不完全一致。
6. 后台轮询会增加一定耗电，因此已放到设置页面作为开关。

## 八、后续建议

- 把关键状态判断逻辑拆成独立类，降低 `MainActivity` 体积。
- 将接口请求封装成统一 API 层，便于维护和异常处理。
- 增加正式 release 签名包构建流程。
- 增加日志开关，方便现场排查账号过期、告警不响、压力不更新等问题。
- 对静止压力采集策略增加本地调试面板，显示每个传感器的当前状态、稳定计时、锁定原因。
- 对报警声音、后台轮询、角标同步做更多真机兼容测试。
