### 静止压力旧值自动重新采集

- 修复传感器静止压力曾采集为 `0` 后，即使后续充压也一直锁定为 `0` 的问题。
- 离线传感器当前压力与已锁定静止压力明显不同时，自动开启新一轮 3 分钟稳定采集，不再需要清除 App 缓存。
- 新值稳定满 3 分钟后才替换旧静止压力；采集期间继续显示旧值，避免动态压力直接覆盖。
- 在线模具与离线模具统一复用静止压力采集初始化逻辑。
- 影响文件：`app/src/main/java/cn/mitebo/iot/MainActivity.java`。

### 零压力传感器下限静音按钮

- 在模具压力上下限弹窗左下角增加喇叭图标按钮。
- 当前选中传感器实时压力为 0 时，可将其报警下限切换为 0；其他传感器不受影响。
- 离线模具内喇叭按钮始终可操作；在线模具仍仅允许界面显示为 0 的传感器归零。
- 零压力判断与界面两位小数显示保持一致，避免微小残余值导致按钮错误变灰。
- 下限归零后显示静音喇叭图标，再次点击可恢复归零前的下限值和正常喇叭图标。
- 原下限按账号和传感器保存，保存归零后重新打开弹窗仍可恢复。
- 原下限以后台压力单位保存，切换 App 压力单位后恢复值仍能正确换算。
- 新增资源：`ic_volume_on.xml`、`ic_volume_off.xml`。
- 喇叭资源按参考图调整为青蓝圆形底、白色线框，静音状态使用白色斜线，并去除按钮方框背景。
- 喇叭按钮背景改为与 App 主操作按钮一致的蓝色到青色渐变。
- 影响文件：`app/src/main/java/cn/mitebo/iot/MainActivity.java`。
- 验证：Android 资源解析和 Debug Java 编译通过。

### 在线模具静止压力独立采集

- 修复模具整体处于在线状态时，所有传感器的静止压力计时被每次刷新清空的问题。
- 在线模具改为按传感器分别判断压力波动和连续稳定时间。
- 同一模具内部分传感器压力为 0 或仍在波动时，不再阻断其他有压力传感器采集静止压力。
- 单个传感器发生超过稳定容差的波动时，只重新计算该传感器自己的 3 分钟稳定时间。
- 已锁定的静止压力继续保留，新的候选压力连续稳定 3 分钟后才更新。
- 影响文件：`app/src/main/java/cn/mitebo/iot/MainActivity.java`。
- 验证：`git diff --check` 和 Debug Java 编译通过。

## 2026-07-01

### 后台告警轮询稳定性

- 后台告警请求改为上一轮完成后再安排下一轮，避免网络较慢时多个轮询请求重叠。
- 增加请求运行状态和轮询代次校验，切换账号或停止服务后丢弃旧请求结果。
- 接口请求失败时保留现有告警状态，不再误清空角标和正在告警状态。
- 后台监控关闭、账号失效、版本过期或前台服务超时时，统一停止轮询、铃声和通知。
- 后台告警服务只拉取告警列表，不刷新设备压力，避免影响静止压力采集。
- 影响文件：`app/src/main/java/cn/mitebo/iot/AlarmMonitorService.java`。

### 模具状态与静止压力锁定

- 区分模具列表状态和设备内嵌模具状态，避免把设备记录启用状态误判为模具生产状态。
- 无法确认新状态时延用上一次状态，避免短暂缺少字段导致在线模具误下线。
- 静止压力锁定状态和下一轮待采集状态写入本地存储，App 重启后继续保持。
- 只有模具经历在线生产并再次下线后，才允许开启下一轮静止压力采集。
- 刷新接口状态不明确时不再用实时动态压力覆盖已锁定的静止压力。
- 影响文件：`app/src/main/java/cn/mitebo/iot/MainActivity.java`。

### 设备与网关界面

- 设备卡片增加 `MAC：` 标签，并将在线状态胶囊放到 MAC 地址同一行。
- 收窄设备卡片右侧压力区域，为 MAC 地址和状态留出更多空间。
- 网关卡片操作按钮去除文字图标、缩小宽度并增加中间间距。
- 登录账号下拉箭头适当放大；设置页压力单位说明改为“切换压力单位”。
- 影响文件：`app/src/main/java/cn/mitebo/iot/MainActivity.java`。

- 将传感器静止压力的连续稳定判定时间从 10 秒调整为 3 分钟。

- 采集成功后继续锁定静止压力，直到模具再次上线并进入下一次下线采集周期。

- 网关绑定关系按分页完整读取；绑定关系加载失败时保留原候选列表，避免误清空。
